### What I learned
This is the test, where I want to try to learn about the usage of Svelte and how it works. As it's not SvelteKit, so not the framework, but just a fundamental part of it - I wanted to try learning from basics. 

### Continued development

Maybe I'd like to explore more about the challenge, about the Svelte, but I'm not sure. It was indeed an interesting challenge. I've spent around 1.5 or 2 hours implementing static pages and then who knows how much time developing logic. I needed to motivate myself over and over again, but I guess I did it and I have my first challenge and my first Svelte project. Next time I will learn about the technology before, or use step-by-step tutorials. This time it was too chaotic, and I didn't know where to start with implementation or how two-way data binding works. I need to come back to this after I learn more about Svelte and SvelteKit. 

### AI Collaboration
One of the reasons to do challenges is to stop brainrot from using AI, so no AI didn't code me, but I did ask for it to be broken down the whole challenge into subtasks and tasks, as I didn’t know where to start. I admit that I turned to ChatGPT when looking for answers to questions about bugs or things I didn’t understand in Svelte. That’s precisely why you need to get to grips with the technology first, rather than trying to figure out how to do things as you go along. That said, asking ChatGPT for advice these days is like searching on Google or Stack Overflow. I’ve used both of those resources as well, so shame on me for using tools like AI for learning. 
At the end of the day, the AI didn’t do the work for me, and I tried to use it as little as possible, relying more on my own brain. Judging by the quality of the code, it’s hard to tell whether it’s me who’s so "clever" or the AI.
